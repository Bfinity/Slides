Semantic Tags

  • Elements with a meaning
  • Describes its meaning to the browser and developer
  • Useful for text-to-speech
  • Makes types, hierarchies and relationships clearer
  • HTML5 has new semantic tags that eliminate the need for previous semantic tags.

...and Iterative Development

Iterative Development

Why?

  • Because waiting to find bugs until we are done writing all our code delays deployment

How?

Get into the habit of iterating through three steps as you're developing:

  1. Write some code
  2. Test your code
  3. Fix any bugs

Iterative Development

There are other software development methodologies that development teams use but most have this short iterative process (in some fashion) in common.

<strong> </strong>

  • Use CSS to add additional effects

<blockquote> </blockquote>

  • Specifies section quoted
  • Usually renders on browsers as indented text
  • In HTML4.01, defines a long quote
  • In HTML5, defines a quote from another source
<blockquote>
With attributes like elegant, creative, long-termed, and seeing the
forest and the trees, our results indicate that good software
engineering requires engineers to make complex, experienced-driven,
contextual considerations.
</blockquote>

<cite> </cite>

  • Citation (HTML4.01)
  • HTML5 defines this as the title of the quote/work.
  • Renders on the browser as italicized text
  • Can be added to the head tag of a blockqote, but it won't be visible on the page.
<cite>
https://faculty.washington.edu/ajko/papers/Li2015GreatEngineers.pdf
</cite>

<q> </q>

  • Defines a short quote
  • Renders on the browser as quotation marks
  • Note: Internet Explorer does not render this

What makes a great software engineer?
</q>

<abbr> </abbr>

  • Defines an abbreviation
  • Renders on the browser as the abbreviation
  • Full title displays when you mouse over it
  • Needs a title attribute

DL
</abbr>

<dfn> </dfn>

  • Defines a definition
  • Renders on browsers as italicized text
  • Parent tag of the <dfn> must also contain the definition inside it
  • Many ways to use this tag
<p> <dfn>Developer</dfn>
an artist and craftsperson of software</p>

<address> </address>

  • Defines the contact information of the author/owner
  • Not for the describption of a postal address
  • Renders on browsers as italicized block
  • Usually included in the <footer> tag
<footer>
<address>Written By: (your name)
Visit us at: the World Wide Web
</address>
</footer>

<del> </del>

  • Defines deleted text
  • Renders on the browser as strike through
  • Used with the <ins> tag (see next slide)

<ins> </ins>

  • Defines inserted text
  • Renders on the browser as underlined
  • Used with the <del> tag

<p>Coding can be <del>too hard</del>
<ins>a great challenge</ins></p>

<s> </s>

  • Defines text that is no longer correct/accurate/relevant but should not be deleted
  • Also renders on browsers as strike through
  • This was redefined by HTML5 (used to be defined as strike-through text)
<p><s>Development takes a lot of math skills</s></p>
