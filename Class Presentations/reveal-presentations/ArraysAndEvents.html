More information about arrays

Every array has a property called length, which shows you how many elements it contains.

This property doesn't play any tricks like the array indices do. The following array has a length of three (sounds.length === 3).

``` var sounds = ["zip", "zap", "zop"]; ```

Arrays also have methods that allow you to find elements in them, combine arrays, add new elements, return elements, etc.

Event listeners

JavaScript allows you to update your webpage by responding to certain things that happen to HTML elements. These things are called "events", and they can be initiated by the user or they can be automatic.

There are three different ways to "register event handlers", or prepare an HTML element to respond to an event:

  • inline, in HTML (ancient)
  • traditional: uses JavaScript
  • W3C method: event handling

Source: http://www.quirksmode.org/js/introevents.html

inline, in HTML (ancient)

Inline event handlers are the oldest way to add events to HTML elements.

They are frowned upon today because they require mixing webpage action with presentation instead of separating them into different classes.

However, we'll go over this anyway, just so you're familiar .

``` <h3 onclick="alert('I\'ve been clicked!')">Challenges</h3> <a href="www.facebook.com" onclick="doSomething()">Link to FB</a> ```

traditional: uses JavaScript

This method is considered cleaner than the inline HTML way.

All modern browsers can read and understand this way of registering event handlers.

``` HTML: <h3 id="challenges">Challenges</h3> JS: var challenges = document.getElementById('challenges'); challenges.onclick = doSomething; ```
    Note:
  1. element property is all lowercase;
  2. function doesn't have parentheses after it. (If it did, the function would execute right then and there, which is not what we want.)
  3. We can add as many functions to an event as we want to.

W3C method: event handling

This is another clean way of registering event handlers. It lines up with the W3C (World Wide Web Consortium) standard, but that doesn't work well on all browsers, unfortunately.

``` HTML: <h3 id="challenges">Challenges</h3> JS: var challenges = document.getElementById('challenges'); challenges.addEventListener('click', doSomething, false); ```
    Note:
  1. No need for 'onClick', it's just 'click'
  2. Again, the function doesn't have parentheses after it.
  3. We can add as many event listeners as we want to.

The third parameter, the boolean, has to do with the question of what to do if an element and its "ancestor" need to respond to the same event--which one should respond first?

(Imagine a word in a three-word heading, for example, or any other element inside an element, and both have an onclick event handler.)

true: the outer element will respond before the inner element. This is called event capturing.

false: the inner element will respond before the outer element. This is called event bubbling.

false is the default.
